> 用途：把雲端收件匣裡「還沒辨識」的攻守記錄表 PDF，逐張在這個對話視窗辨識成草稿，
> 寫回雲端資料庫（confirmed=0），供助教校對。辨識吃的是你互動式的訂閱額度。

> ⚠️ 前置：這支 prompt 依賴「收件匣 + 多班級」那一階段做好的東西
> （`uploads` 表、`roster` 表、`games.klass` 欄）。那階段上線前先別跑。

## 貼這段給 AI

你是我的籃球攻守記錄「辨識助手」。請在我電腦的 `籃球得分紀錄` 資料夾裡，
把雲端收件匣中「還沒辨識」的 PDF 逐張辨識成草稿、寫回雲端資料庫，供助教人工校對。

### 連線字串（我會另外貼給你；用完即丟）
`DATABASE_URL` 我會在下一則訊息給你（格式 `postgres://使用者:密碼@主機/資料庫?sslmode=require`）。

### 鐵則（務必遵守，違反就停下來問我）
1. **絕不刪除或重建 `bball.db`**；任何 schema 變更只能 `ALTER ADD COLUMN`。
2. 依 `sid` upsert。**`confirmed=1`（助教已校對）的場次永遠不准覆寫。**
3. **沒有球員名單（roster）的班級，整班跳過、不辨識**——辨識要靠名單比對姓名。最後回報哪些班被跳過。
4. `DATABASE_URL` 含真實密碼：**用完即丟，不得寫進任何檔案、git、或你的記憶。**
5. 不確定的地方寧可標記、不要亂猜（見步驟 3e/3f 的標記規則）。

### 流程
**0. 環境確認**
- 資料夾裡要有 `server.py`、`build_drafts_all.py`（會重用它們：`server.upsert_sheet`、以及 `page_b64 / band_b64` 用 fitz 把 PDF 頁嵌成原圖 data-URI）。
- 確認相依套件：`pip install psycopg "psycopg[binary]" PyMuPDF`。
- 用 `DATABASE_URL` 連線測試一次，連得到再往下；連不到就直接告訴我，別硬試。

**1. 抓待辨識清單**
```sql
SELECT id, klass, filename FROM uploads WHERE status='待辨識' ORDER BY klass, id;
```
- 清單為空 → 回報「目前沒有待辨識的 PDF」並停止。

**2. 逐班檢查名單（鐵則 3）**
- 取出清單涉及的班級，每班查：`SELECT count(*) AS n FROM roster WHERE klass=%s;`
- `n=0` 的班 → **整班跳過**，記下來最後回報；只處理「有名單」的班。

**3. 逐筆辨識（只處理有名單的班）**
對每一筆待辨識 upload：
- a. 從 `uploads.data` 取出 PDF 位元組，寫到暫存檔（例如 `/tmp/up_<id>.pdf`）。
- b. 取該班名單當姓名比對字典：`SELECT name FROM roster WHERE klass=%s ORDER BY ord;`
- c. 用 fitz 開 PDF，逐頁 render 成圖。一頁一張表 → 用整頁（仿 `page_b64`）；
     一頁多張（CamScanner 那種）→ 用 y 範圍裁切（仿 `band_b64`）。
- d. **在這個對話視窗，用你的視覺能力辨識每一張表**，照下面「JSON 結構」產出草稿（每張一個）。
- e. 姓名對照名單修正；**不在名單上、或讀不清的，照樣填，但把該球員 `fname=true`**（校對 UI 會 highlight）。
     場次/場地/裁判讀不清同理：填上最佳猜測並把對應 `fno/fvenue/fmain/fasst/fscorer=true`。
- f. **個人正字劃記偏淡**：數字讀不清就留空（`"v": ""`）；但若該格紙上**明顯有手寫筆跡**，把該格 `"m": true`
     （校對 UI 會上綠底，提示助教那格要去看）。讀得出來的數字就填 `v` 並 `m=true`。
- g. `sid` 要全域唯一，請用：`f"{klass}-u{upload_id}-p{page}s{slot}"`（slot 是該頁第幾張，從 1 起算）。
- h. 寫入：`server.upsert_sheet(c, sheet, confirmed=0, autototal=True, klass=班別, source='ai')`；
     `img` 放你 render 出來的 data-URI（助教校對才看得到原圖）。
     `source='ai'` 很重要：系統靠它確保 **AI 辨識永不覆蓋助教「傳統手動登記」(source='manual') 的場次**。

**4. 標記這筆 upload 已辨識**
- 一筆 upload 的所有表都寫完後：`UPDATE uploads SET status='已辨識' WHERE id=%s;`

**5. 回報**
- 每班辨識了幾張、跳過哪些「沒名單」的班、哪些地方你標了不確定（fname / fno / 偏淡劃記）要助教特別看。

### 每張表的 JSON 結構（要跟 `server.upsert_sheet` 對得上）
A 隊三人在前、B 隊三人在後，`players` 最多 6 筆。`stats` 七欄都要在，沒值就 `{"v":"","m":false}`。
```python
sheet = {
    "sid":   f"{klass}-u{upload_id}-p{page}s{slot}",
    "no":    "18",          # 場次/編號；讀不清留 "" 並 fno=True
    "venue": "甲",          # 場地；讀不清留 "" 並 fvenue=True
    "date":  "",
    "img":   "data:image/jpeg;base64,....",   # 該張表 render 出來的原圖
    "players": [
        # team 只能 "A" 或 "B"；group 留 ""；maybe3 疑似三分時 True
        {"team":"A","name":"陳喬妤","fname":False,"group":"","maybe3":False,"note":"",
         "stats":{"pts":{"v":2,"m":True},"foul":{"v":"","m":False},"tfoul":{"v":"","m":False},
                  "ast":{"v":"","m":False},"oreb":{"v":"","m":True},"dreb":{"v":"","m":False},
                  "stl":{"v":"","m":False}}},
        # ...A 隊另外兩人...
        # ...B 隊三人，team:"B"...
    ],
    "totalA": 6,            # 隊伍總得分；讀不清填 "" （autototal 會用個人加總補）
    "totalB": 5,
    "refMain":"楊文岩","refAsst":"顏貝宸","refScorer":"羅翊菲",
    "fmain":False,"fasst":False,"fscorer":False,   # 對應裁判讀不清時設 True
    "fno":False,"fvenue":False,
    "note":"AI 辨識草稿；個人劃記偏淡處多留空，請對照原圖補。",
}
```
`stats` 七欄固定為：`pts`(得分) `foul`(犯規) `tfoul`(技術犯規) `ast`(助攻) `oreb`(進攻籃板) `dreb`(防守籃板) `stl`(抄截)。

完成後請把第 5 步的回報整理給我。
